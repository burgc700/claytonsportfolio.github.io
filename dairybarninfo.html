About the project

School project to help develop Object Oriented Principles for a program. Created as a knock off menu from Diary Queen that has burgers, ice cream, and drink options. Starting on the main WPF layout we see the menu items and the menu.

Once the user finds an item the want they can select that item. It will then be added to the order on the left side of the screen and all the other fields will update. The user can also change the options of that menu item to customize their order.

There is a deal option for burgers and ice cream that allows the user to get 25% off when purchasing both items together. You can select both options and customize both options. For both regular items and the pick two deal can be customized from the menu screen and removed from the order

After the user selects everything for order they can select the Complete order button at the bottom. They will be redirected to another screen to input the amount they are paying for the order. If the amount is less than the total red appears and the button is not active, if the amount is greater than or equal to the total the button is active and the user can check out.

After the prompt is cleared the UI resets and the order number is incremented by one. The receipts for the orders are located in the /bin folder if you follow the path all the way back to the receipt.txt file.

The webpage for this project is simple and just shows all possible options of all menu items, some ingredient information, and a place to search and sort based off price and calories. Selecting burger, ice cream, or drinks from the top will filter the page down to that section of menu items.

The user can select a range or a maximum and minimum for calories and price then all products will be sorted to fit that criteria.

There is also a page for reviews so a customer can leave a review of the fast food joint when the are done. Complimented by a date and time stamp.
