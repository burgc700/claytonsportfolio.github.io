About the project

This was a group project to connect everything we've learned through out the whole semester about database management. Throughout this project my main focus was the front-end implementation. After all that it led to diagnosing bugs as well. The main UI is pretty simple and straight forward for simplicity.

Each drop down has two different UIs one to add and one to edit customers, employees, and projects. While the buttons on the right are buttons to activate some aggregating queries.

So if we wanted to add a customer we would click customer, add customer then we would get the following prompt to fill out and click ok.

Then after it will show all the the customers in order by customerID. The edit feature is set up the same way. Select customer, edit customer, then type in the customer id whose information has changed. You have to type in everything again which is a downfall but part of my improvements are to auto populate the data from the ID and allow the user to change what they need to change without typing all the info in again.

The updated information does show up though to reflect the changes. The last thing is the aggregating queries. These all have parameters you can set to get the information from a specific project or from a given time frame.

So this query shows the total hours on a given project. All other queries do something similar in nature showing more overall company statistics than individual person information. You can also search for customers, employees and projects based off their id or name.
